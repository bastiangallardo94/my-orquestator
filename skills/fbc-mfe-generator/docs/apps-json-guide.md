# apps.json - App Shell Registration Guide

## Overview

Every microfrontend generated by this skill includes an `apps.json` file at the project root. This file contains the configuration needed to register the MFE in the portal's app-shell.

## Purpose

The `apps.json` file serves as a **template** for registering your microfrontend in the main portal. The app-shell uses this configuration to:

1. **Discover** your MFE's remote entry point
2. **Load** your MFE dynamically via Module Federation
3. **Route** requests to your MFE based on path patterns
4. **Mount** your MFE when the user navigates to its routes

## File Structure

```json
[
  {
    "appName": "importInspections",
    "componentImport": "importInspections/App",
    "routes": "['/foreign-trade/inspections']",
    "show": "showWhenPrefix",
    "appRemoteName": "importInspections",
    "remote": "http://localhost:8000/remoteEntry.js"
  }
]
```

## Field Definitions

### `appName`
- **Type**: `string`
- **Description**: Unique identifier for your microfrontend
- **Value**: Same as your `APP_NAME` configuration
- **Example**: `"importInspections"`, `"importMaintainersRouter"`

### `componentImport`
- **Type**: `string`
- **Description**: Module Federation import path
- **Format**: `{appName}/App`
- **Example**: `"importInspections/App"`
- **Note**: Must match the `exposes` configuration in your `module-federation.config.js`

### `routes`
- **Type**: `string` (array stringified)
- **Description**: Array of route patterns that trigger this MFE
- **Value**: Same as your `APP_PATH` wrapped in an array
- **Example**: `"['/foreign-trade/inspections']"`
- **Multiple routes**: `"['/foreign-trade/inspections', '/ft/inspections']"`

### `show`
- **Type**: `string`
- **Description**: Strategy for when to display/mount this MFE
- **Values**:
  - `"showWhenPrefix"`: Mount when route starts with any route in `routes`
  - `"showWhenExact"`: Mount only on exact route match
- **Default**: `"showWhenPrefix"` (recommended)

### `appRemoteName`
- **Type**: `string`
- **Description**: Name of the Module Federation remote
- **Value**: Same as `appName`
- **Example**: `"importInspections"`

### `remote`
- **Type**: `string`
- **Description**: URL to the `remoteEntry.js` file
- **Format**: `http://localhost:{PORT}/remoteEntry.js`
- **Default**: Port `8000` (placeholder)
- **Production**: Replace with actual deployment URL

## How to Use

### Step 1: Generate Your MFE

When you generate a microfrontend using this skill, an `apps.json` is automatically created with placeholders already replaced.

### Step 2: Review Configuration

Open `apps.json` in your generated project and verify:

```json
[
  {
    "appName": "importInspections",          // ✅ Your APP_NAME
    "componentImport": "importInspections/App", // ✅ Matches exposes
    "routes": "['/foreign-trade/inspections']", // ✅ Your APP_PATH
    "show": "showWhenPrefix",                   // ✅ Standard strategy
    "appRemoteName": "importInspections",       // ✅ Same as appName
    "remote": "http://localhost:8000/remoteEntry.js" // ⚠️ Update port!
  }
]
```

### Step 3: Update Port

Change the `remote` port from `8000` to your actual development port:

```json
{
  "remote": "http://localhost:8500/remoteEntry.js"  // Your APP_PORT
}
```

### Step 4: Register in App Shell

#### Local Development

1. Navigate to the portal's app-shell:
   ```bash
   cd packages/mrch.frontend.cross.app-shell
   ```

2. Open `apps.json`:
   ```bash
   code apps.json
   ```

3. Add your MFE configuration to the array:
   ```json
   [
     {
       "appName": "existingMfe",
       "componentImport": "existingMfe/App",
       "routes": "['/existing-path']",
       "show": "showWhenPrefix",
       "appRemoteName": "existingMfe",
       "remote": "http://localhost:8400/remoteEntry.js"
     },
     {
       "appName": "importInspections",
       "componentImport": "importInspections/App",
       "routes": "['/foreign-trade/inspections']",
       "show": "showWhenPrefix",
       "appRemoteName": "importInspections",
       "remote": "http://localhost:8500/remoteEntry.js"
     }
   ]
   ```

4. Restart the app-shell:
   ```bash
   bun start
   ```

#### Production

For production, update the `remote` URL to your deployment endpoint:

```json
{
  "remote": "https://prod.fbusinesscenter.com/foreign-trade/inspections/remoteEntry.js"
}
```

## Common Issues

### Issue: MFE not loading

**Cause**: Port mismatch  
**Solution**: Verify `remote` port matches your MFE's running port

**Cause**: Component import mismatch  
**Solution**: Verify `componentImport` matches your `module-federation.config.js` `exposes`

### Issue: Route not matching

**Cause**: `routes` doesn't match navigation path  
**Solution**: Ensure `routes` includes the correct path patterns

**Cause**: Wrong `show` strategy  
**Solution**: Use `showWhenPrefix` for route prefixes, `showWhenExact` for exact matches

### Issue: Multiple MFEs conflict

**Cause**: Overlapping `routes`  
**Solution**: Ensure each MFE has unique route patterns

## Advanced Usage

### Multiple Routes

To handle multiple route patterns:

```json
{
  "routes": "['/foreign-trade/inspections', '/ft/inspections', '/inspections']"
}
```

### Environment-Specific Configuration

Use environment variables in your CI/CD to replace the `remote` URL:

```bash
# Development
remote: "http://localhost:8500/remoteEntry.js"

# Staging
remote: "https://staging.fbusinesscenter.com/foreign-trade/inspections/remoteEntry.js"

# Production
remote: "https://prod.fbusinesscenter.com/foreign-trade/inspections/remoteEntry.js"
```

### Nginx Routing

Ensure your Nginx configuration routes to the correct MFE:

```nginx
location /foreign-trade/inspections {
    proxy_pass http://localhost:8500;
}
```
